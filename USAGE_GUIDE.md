## 🚀 使用步骤

### 1. 部署后端服务

```bash
# 进入后端目录
cd backend

# 部署到Cloudflare Workers
wrangler deploy
```

部署成功后,会得到一个Workers URL,例如:
`https://accounting-exam-api.1227944456.workers.dev`

### 2. 配置API地址

打开主HTML文件,找到以下代码并修改API_BASE_URL:

```javascript
const API_BASE_URL = 'https://accounting-exam-api.1227944456.workers.dev';
```

将其修改为您的实际Workers URL。

### 3. 启动使用

直接在浏览器中打开主HTML文件即可:
```

```

## ✨ 新功能说明

### 1. 强制登录验证
- 未登录用户访问主页面,自动跳转到登录页面
- 已登录用户访问登录页面,自动跳转到主页面
- Token过期时,自动跳转到登录页面并提示

### 2. 个人中心
点击右上角的用户头像,可以:
- 查看用户信息(用户名、ID)
- 修改密码
- 切换账号
- 退出登录

### 3. 实时数据同步
- 学习进度修改后立即同步到服务器
- 错题添加/删除后立即同步到服务器
- 页面刷新时从服务器加载最新数据
- 支持多设备同时在线,数据实时一致

### 4. 网络状态监测
- 离线时显示全屏提示,禁止所有操作
- 网络恢复后自动刷新页面
- 提供手动刷新按钮

### 5. Token自动刷新
- Token剩余有效期<25%时自动刷新
- 刷新失败自动跳转登录页面
- 无需手动管理Token

## ⚠️ 注意事项

1. **网络依赖**: 系统现在完全依赖网络,离线时无法使用
2. **数据安全**: 所有数据存储在服务器,请定期备份
3. **Token管理**: Token自动刷新,无需手动处理
4. **首次使用**: 如果localStorage有历史数据,建议先迁移到服务器

## 🐛 常见问题

### Q1: 登录后页面空白?
**A**: 检查API_BASE_URL是否正确配置,确保Workers已成功部署。

### Q2: 数据同步失败?
**A**: 检查网络连接,查看浏览器控制台错误信息。

### Q3: Token过期频繁?
**A**: Token有效期1小时,会自动刷新。如果频繁过期,检查系统时间是否正确。

### Q4: 离线时如何处理?
**A**: 系统会显示全屏提示,禁止操作。网络恢复后自动刷新。

### Q5: 如何迁移历史数据?
**A**: 首次登录时,如果localStorage有数据,需要手动迁移到服务器(功能待实现)。
